# SEO: правила и рекомендации для сайта Siberian Motorbears

Документ описывает стандарты SEO для siberian-motorbears.ru. Следовать им при доработке существующих страниц и при добавлении новых.

---

## 1. Техническая база

### Sitemap и robots
- **sitemap.xml** — в корне сайта. Все значимые страницы с полным URL (`https://siberian-motorbears.ru/...`), для каждой: `loc`, `lastmod` (дата в формате YYYY-MM-DD), `changefreq`, `priority`.
- **robots.txt** — в корне. Строка `Sitemap: https://siberian-motorbears.ru/sitemap.xml`, при необходимости правила `User-agent` и `Allow`/`Disallow`.
- При появлении новых страниц — добавлять их в sitemap. При изменении контента — обновлять `lastmod` у соответствующих URL.

### Canonical
- На **каждой** странице в `<head>` одна каноническая ссылка:
  - главная: `https://siberian-motorbears.ru/`
  - внутренние: `https://siberian-motorbears.ru/раздел/` (со слэшем в конце, без `index.html`).
- Формат canonical единый по всему сайту.

## 2. Мета-теги и превью

### Обязательно на каждой странице
- **`<title>`** — уникальный, до ~60 символов. Ключевая фраза + гео/оффер при необходимости + « | Siberian motorbears».
- **`<meta name="description">`** — уникальный, 150–160 символов. Суть страницы, ключевые слова, призыв (позвонить, заказать, забронировать).
- **`<meta name="keywords">`** — релевантные запросы через запятую (в т.ч. гео: Новосибирск, Алтай, Байкал, Сибирь где уместно).
- **Open Graph**: `og:type`, `og:url`, `og:title`, `og:description`, `og:image`, `og:locale` (ru_RU). Текст — как в title/description страницы; картинка — одна на сайт или своя для товара/услуги.
- **Twitter Card**: `twitter:card` (summary_large_image), `twitter:title`, `twitter:description`, `twitter:image` — в том же духе, что и OG.

### Изображения
- У **всех** `<img>` осмысленный атрибут **`alt`**: контекст страницы + содержание картинки (модель, сцена, партнёр и т.д.). Пустых `alt=""` избегать.

## 3. Структурированные данные (JSON-LD)

### На всех страницах
- **LocalBusiness** (или Organization): название, url, logo, telephone, email, address (Кольцово, Новосибирская обл., д. 6А), sameAs (VK, Telegram).

### На страницах товаров/услуг
- **Product**: name, description, image (полный URL), brand (Siberian motorbears), offers (url, priceCurrency: RUB, price при известной цене). Тексты и цены — как на странице.

### На страницах с иерархией
- **BreadcrumbList**: itemListElement с полными URL. Порядок: Главная → раздел → подраздел (текущая страница последней, без ссылки в разметке не обязательно, но URL указать).

### На страницах с блоком «Частые вопросы»
- **FAQPage**: mainEntity — массив Question с name (вопрос) и acceptedAnswer (Answer с text). Текст ответов — без HTML, в одном формате с видимым контентом.

## 4. Контент и структура страниц

### Заголовки
- На странице один **H1** — главная тема (услуга, модель, раздел).
- Далее **H2** для блоков, **H3** для подпунктов. Иерархия без пропусков (не H1 → H4).

### Главная страница
- В первом экране или сразу под H1 — **лид** с явным оффером и гео: производство/прокат, Новосибирск, купить/арендовать, Алтай, Байкал, Сибирь.
- Цены — видимы (в кнопках или отдельной строкой) для основных моделей/услуг.
- Призыв к действию (аренда/покупка/контакты) — явный и заметный.

### Страницы аренды/производства
- Чётко указаны цены или «от X ₽» где применимо.
- Контакты (телефон, почта, мессенджеры) — легко найти и кликнуть.

### Гео и ключевые фразы
- В title/description/keywords и в тексте уместно использовать: Новосибирск, Кольцово, Алтай, Горный Алтай, Байкал, Сибирь, НСО.
- Для аренды: «прокат», «аренда», «от X ₽/сутки». Для производства: «производство под заказ», «от X ₽», сроки при возможности.

---

## 5. Навигация и ссылки

### Хлебные крошки
- На **всех внутренних** страницах (кроме главной) — блок breadcrumbs в начале `<main>`.
- Разметка: `BreadcrumbList` (JSON-LD). Визуально: выравнивание по правому краю (как в текущей вёрстке).
- Цепочка: Главная → раздел (Аренда/Производство/Контакты/Реквизиты) → при необходимости подраздел (модель, страница).

### Внутренние ссылки
- Из главной — на разделы аренды, производства, контактов и ключевые страницы (модели, примеры).
- В меню и футере — стабильные ссылки на основные разделы.

## 6. Чек-лист для новой страницы

При добавлении новой страницы проверить:

- [ ] URL добавлен в **sitemap.xml** (loc, lastmod, changefreq, priority).
- [ ] В `<head>`: **canonical**, **title**, **description**, **keywords**.
- [ ] В `<head>`: **Open Graph** и **Twitter Card** (title, description, image, url).
- [ ] На странице — разметка **LocalBusiness** (как на остальных).
- [ ] Если страница товара/услуги — разметка **Product** с ценой/оффером.
- [ ] **Breadcrumbs** (HTML + BreadcrumbList) с корректной цепочкой.
- [ ] Один **H1**, логичная иерархия H2/H3.
- [ ] У всех изображений заполнен **alt**.
- [ ] При наличии блока FAQ — разметка **FAQPage** и совпадение текстов с видимым контентом.

---

## 7. Дополнительные рекомендации

- **Скорость**: по возможности оптимизировать размер изображений (сжатие, при необходимости WebP), сохранять `loading="lazy"` для картинок ниже первого экрана.
- **Мобильная вёрстка**: корректное отображение и читаемость на телефонах.
- **Контент**: при расширении сайта полезны блоки FAQ на тематических страницах, отзывы (с именами/датами), уникальные тексты без копипаста с других страниц.
- **Единообразие**: стиль title/description (бренд в конце, ключевые фразы в начале), формат цен (от X ₽/сутки, от X ₽), формат телефона (+7 (913) 460-20-50) — держать одинаковым по сайту.
