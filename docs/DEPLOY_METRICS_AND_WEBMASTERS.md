# Сохранение Метрики и вебмастеров при обновлении сайта

Домен остаётся тем же (siberian-motorbears.ru). Чтобы после выкладки новой версии сайта продолжали работать **Яндекс.Метрика**, **Яндекс.Вебмастер**, **Google Search Console** и **VK Пиксель**, в новой вёрстке должны быть те же коды и проверочные метки.

**В текущей вёрстке (папка `site/`) на всех 11 страницах уже подставлены: номер счётчика Яндекс.Метрики 66322963 и VK Пиксель (Top.Mail.Ru) с кодом 3727349.** (взят с живого сайта siberian-motorbears.ru). Мета-теги проверки Яндекс.Вебмастер и Google Search Console на текущем сайте в HTML не найдены — скорее всего, проверка сделана через DNS или через файл в корне. Если у тебя в кабинетах указан способ «Мета-тег», впиши коды в `content` у соответствующих тегов; если проверка по DNS или по файлу — оставь `content=""` пустым или удали эти теги, главное не удалять файлы проверки из корня при выкладке.

## Что сделать до или сразу после выкладки

### 1. Яндекс.Метрика

- На **текущем живом сайте** открой код страницы (ПКМ → «Просмотр кода страницы») и найди блок с `mc.yandex.ru` или `metrika.yandex` (счётчик с номером вида `ym(XXXXXX, "init", ...)`).
- Скопируй **целиком** этот блок (обычно два фрагмента: один в `<head>`, один перед `</body>`).
- В **новой версии** сайта вставь тот же код:
  - фрагмент для `<head>` — в `<head>` каждой страницы (или в общий шаблон/компонент, если сайт собирается из шаблонов);
  - фрагмент перед `</body>` — перед закрывающим тегом `</body>` на каждой странице.
- Если новая версия — статичные HTML в папке `site/`, код нужно добавить во все 11 файлов: `index.html`, `order/index.html`, `rent/index.html`, `contact/index.html`, `blog/index.html`, `legal/index.html`, `examples/index.html`, `baribal/index.html`, `panda/index.html`, `baron/index.html`, `mia/index.html`.

**Итог:** на новом сайте должен быть тот же счётчик Метрики с тем же ID — тогда статистика и цели продолжат работать без перенастройки в интерфейсе.

---

### 2. Яндекс.Вебмастер

- Зайди в [Яндекс.Вебмастер](https://webmaster.yandex.ru/) → свой сайт → раздел про проверку прав (например, «Подтверждение прав»).
- Посмотри, **как** сейчас подтверждён сайт:
  - **Мета-тег** — в `<head>` текущего сайта есть строка вида  
    `<meta name="yandex-verification" content="XXXXXXXXXXXXXXXX" />`  
    Эту **строку целиком** нужно добавить в `<head>` каждой страницы новой версии (в тот же список, что и в п.1).
  - **Файл проверки** — в корне сайта лежит файл вида `yandex_XXXXXXXXXXXXXXXX.html`. Этот файл нужно **скопировать в корень новой версии** (рядом с `index.html` и `sitemap.xml`), чтобы по адресу `https://siberian-motorbears.ru/yandex_XXXXXXXXXXXXXXXX.html` он отдавался и на новом сайте.
  - **DNS** — если проверка только по DNS, ничего в коде менять не нужно; при том же домене проверка сохранится.

**Итог:** способ проверки в новой версии должен быть таким же (тот же мета-тег или тот же файл в корне).

---

### 3. Google Search Console

- Зайди в [Google Search Console](https://search.google.com/search-console) → свой ресурс (siberian-motorbears.ru).
- Открой «Настройки» → «Проверка прав владельца» (или аналог) и посмотри, **какой способ** проверки используется:
  - **Мета-тег HTML** — в коде есть строка вида  
    `<meta name="google-site-verification" content="XXXXXXXXXXXXXXXX" />`  
    Её нужно добавить в `<head>` каждой страницы новой версии.
  - **HTML-файл** — в корне сайта лежит файл вида `googleXXXXXXXXXXXXXXXX.html`. Его нужно **скопировать в корень новой версии**, чтобы он был доступен по тому же URL.
  - **Google Analytics / Google Tag Manager / DNS** — если проверка только через них или DNS, при том же домене и тех же кодах/записях проверка сохранится; при смене вёрстки важно не удалять код Analytics/GTM (см. ниже при необходимости).

**Итог:** тот же способ проверки на новом сайте (тот же мета-тег или тот же файл в корне).

## Краткий чеклист перед выкладкой новой версии

- [ ] Код **Яндекс.Метрики** (с тем же номером счётчика) добавлен во все страницы новой версии.
- [ ] **Яндекс.Вебмастер:** на новом сайте есть тот же мета-тег проверки или тот же файл проверки в корне.
- [ ] **Google Search Console:** на новом сайте есть тот же мета-тег проверки или тот же файл проверки в корне.
- [ ] После выкладки при необходимости проверить в интерфейсах Вебмастера и GSC, что сайт по-прежнему в статусе «Подтверждён».

Если при выкладке не менять домен и не удалять эти коды/файлы, подключение Метрики, Вебмастера и Search Console сохранится.
